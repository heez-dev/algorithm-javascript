# 시간 복잡도 (Time Complexity)

### 시간 복잡도란?

시간 복잡도는 **입력 크기(n)가 증가할 때 알고리즘의 연산 횟수가 얼마나 증가하는지를 나타내는 척도**이다.
실제 실행 시간(초)을 의미하는 것이 아니라 **연산량의 증가율**을 표현한다.

---

### 시간 복잡도를 알아야하는 이유

코딩 테스트에서는 제한된 시간 안에 문제를 해결해야 한다.
같은 문제라도 알고리즘에 따라 실행 시간이 크게 달라질 수 있다.
예를 들어 다음 두 코드는 모두 배열을 순회하지만 시간 복잡도가 다르다.

```JavaScript
// O(n)
for (let i = 0; i < n; i++) {}
```

```JavaScript
// O(n²)
for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {}
}
```

입력 크기가 커질수록 성능 차이는 매우 커진다.

---

### Big-O 표기법

시간 복잡도는 일반적으로 Big-O 표기법을 사용한다.

- 복잡도 순서
  - O(n!) > O(2ⁿ) > O(n³) > O(n²) > O(n log n) > O(n) > O(log n) > O(1)
  - 우측으로 갈수록 복잡도가 낮아 효율적이다.

---

### O(1) - 상수 시간

입력 크기와 관계없이 항상 동일한 연산 횟수를 가진다.

```JavaScript
const arr = [10, 20, 30, 40];
console.log(arr[0]);
```

배열 인덱스 접근은 배열 길이와 관계없이 한 번의 연산으로 수행된다.

```
n = 10
n = 1,000
n = 1,000,000
```

모두 동일한 시간 복잡도를 가진다.

---

### O(log n) - 로그 시간

문제를 절반씩 줄여나가는 알고리즘에서 나타난다.
대표적으로 이진 탐색(Binary Search)이 있다.

```JavaScript
function binarySearch(arr, target) {
    let left = 0;
    let right = arr.length - 1;

    while (left <= right) {
        const mid = Math.floor((left + right) / 2);

        if (arr[mid] === target) return mid;

        if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }

        return -1;
    }
}
```

```
1000 -> 500 -> 250 -> 125 -> ...
```

처럼 탐색 범위를 절반씩 줄인다.

---

### O(n) - 선형 시간

입력 크기만큼 반복한다.

```JavaScript
funciton sum(arr) {
    let total = 0;

    for (const num of arr) {
        total += num;
    }

    return total;
}
```

배열 길이가 증가하면 반복 횟수도 비례해서 증가한다.

```
n = 10 -> 10번
n = 100 -> 100번
n = 1000 -> 1000번
```

---

### O(n log n)

정렬 알고리즘에서 자주 등장한다.

- 대표적인 알고리즘
  - Merge Sort
  - Heap Sort
  - Quick Sort(평균)

```JavaScript
arr.sort((a, b) => a - b);
```

자바스크립트 엔진마다 구현은 다르지만 일반적으로 평균 O(n log n) 수준으로 동작한다.

---

### O(n²)

이중 반복문에서 자주 등장한다.

```JavaScript
for (let i = 0; i < n; i++) {
    for (let j = 0; j < n; j++) {
        console.log(i, j);
    }
}
```

```
n = 10 -> 100
n = 100 -> 10,000
n = 1000 -> 1,000,000
```

입력이 커질수록 연산량이 급격하게 증가한다

---

### O(n³)

삼중 반복문

```JavaScript
for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    for (let k = 0; k < n; k++) {
      console.log(i, j, k);
    }
  }
}
```

```
100 x 100 x 100 = 1,000,000
```

입력이 조금만 커져도 성능 문제가 발생한다.

---

### O(2ⁿ)

모든 경우를 탐색하는 완전 탐색(Brute Force)에서 자주 나타난다.

```JavaScript
function dfs(depth) {
  if (depth === n) return;

  dfs(depth + 1);
  dfs(depth + 1);
}
```

```
n = 10 → 1,024
n = 20 → 1,048,576
n = 30 → 1,073,741,824
```

입력이 조금만 증가해도 연산량이 폭발적으로 증가한다.

---

### 자바스크립트 주요 메서드 시간 복잡도

#### Array

| 메서드     | 시간 복잡도     |
| ---------- | --------------- |
| push()     | 평균 O(1)       |
| pop()      | O(1)            |
| shift()    | O(n)            |
| unshift()  | O(n)            |
| indxOf()   | O(n)            |
| includes() | O(n)            |
| slice()    | O(n)            |
| splice()   | O(n)            |
| sort()     | 평균 O(n log n) |

#### shift() / unshift() 가 O(n)인 이유

```JavaScript
arr.shift();
```

```
[1, 2, 3, 4] -> [2, 3, 4]
```

배열의 첫 요소를 제거하면 나머지 모든 요소를 한 칸씩 앞으로 이동해야 한다.
따라서 배열 길이에 비례하는 시간이 필요하다.

#### Object

| 연산 | 편균 시간 복잡도 |
| ---- | ---------------- |
| 조회 | O(1)             |
| 추가 | O(1)             |
| 수정 | O(1)             |
| 삭제 | O(1)             |

#### Map

| 연산     | 편균 시간 복잡도 |
| -------- | ---------------- |
| set()    | O(1)             |
| get()    | O(1)             |
| has()    | O(1)             |
| delete() | O(1)             |

#### Set

| 연산     | 편균 시간 복잡도 |
| -------- | ---------------- |
| add()    | O(1)             |
| has()    | O(1)             |
| delete() | O(1)             |

---

### 시간 복잡도 별 최대 연산 횟수

| 시간 복잡도 | 최대 연산 횟수  |
| ----------- | --------------- |
| O(n!)       | 10              |
| O(2ⁿ)       | 20 ~ 25         |
| O(n³)       | 200 ~ 300       |
| O(n²)       | 3,000 ~ 5,000   |
| O(n log n)  | 100만           |
| O(n)        | 1,000 ~ 2,000만 |
| O(log n)    | 10억            |

---

### 문제 풀이 전 체크리스트

1. 입력 크기(n)를 확인한다.
2. 허용 가능한 시간 복잡도를 계산한다.
3. 현재 풀이의 시간 복잡도를 계산한다.
4. shift(), unshift() 사용 여부를 확인한다.
5. 탐색이 많다면 Map 또는 Set 활용을 고려한다.
6. 정렬이 필요하다면 O(n log n)을 감안한다.

---

### 핵심 정리

- O(1): 상수 시간
- O(log n): 이진 탐색
- O(n): 순회
- O(n log n): 정렬
- O(n²): 이중 반복문
- O(n³): 삼중 반복문
- O(2ⁿ): 완전 탐색
- 코딩테스트에서는 입력 크기와 시간 복잡도를 먼저 확인하는 습관이 가장 중요하다.
